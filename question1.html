Question 1

For this question, you will make two concurrent calls to an api and perform an action after both have returned. See the api variable in question1.js.

The two calls are api.getUserInfo and api.getUserFriends. Both of them take a callback function which will be asynchronously called with a user info object and a user friends array respectively.

Once the page loads, you should make requests to api.getUserInfo and api.getUserFriends and once both have returned with data, two things needs to happen:

  1. The userData variable (see 'question1.js') should be set to the array [userInfo,userFriends] pair.
  2. Pop up an alert with the user name. Which is the userInfo.name property from api.getUserInfo.

Note 1: You are being evaluated on not just a correct solution, but writing code which has reusable abstractions. Think about a pattern you're doing here you may want to re-use. What if you had to wait for an arbitrary number of calls to return instead of just twox?

Note 2: Do not make serial requests for info and friends.

Bonus: What's the probability the user info comes back before the friends call?
