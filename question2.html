Part 1: Display friends

For this question, you will attach a listener to an API method (see api.listenForFriendChanges in question2.js). Your listener will be called when the user's friends change and it will be called with an array of actions. Each action will be formatted as either an add message: 

		
			{action: 'add', friend:{id:42, name:'Mr. New Friend', strength:0.9}}
Or a remove message: 
		
			{action: 'remove', friend:{id:43, name:'Mr. No longer Friend',strength:0.6}}
While receiving changes from the 'backend', you will be displaying a live updating list of user friend names in the HTML list above ('ul#friends-list'). When a friends gets added, they should be appened to the bottom of the list and should disappear if removed. You should remove the sample friends listed above on document load.

Part 2: Deleting Friends

Let the user delete friends from the list. When a user clicks a name,
  1. Change the color of the name to red to indicate a pending delete OR back to black if it was already red to undo the pending action.
  2. Show or hide the 'Delete Friends' button ('button#delete-button' in the HTML) if there is at least one pending delete or not.
  3. When the user clicks the 'Delete Friends' button, any selected names should be submitted api.submitUserChanges as an array of 'remove' actions like in Part 1. Obviously, the names should be removed from the list. Note that the friend update listener from Part 1 will NOT call you for changes initiated by the user.
Remember, when you submit the users for deletion, you need the whole friend object, not just the name. You must include the id, name, and strength fields you were passed down. Also, remember that the backend can delete a user which has a pending delete. In that case, the user should be immediately removed from the list.
